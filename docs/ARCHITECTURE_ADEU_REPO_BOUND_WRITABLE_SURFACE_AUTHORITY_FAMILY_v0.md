# Architecture: ADEU Repo-Bound Writable-Surface Authority Family v0

Status: architecture / decomposition draft for `V64`.

Authority layer: architecture_decomposition.

This note does not authorize implementation by itself. It defines the bounded family
role that should widen from one exact shipped continuity-backed write exemplar into
one explicit repo-bound writable-surface authority family without letting continuity
state, communication reachability, remote control posture, or prior successful acts
become ambient repo authority.

## 1. Family Role

`V64` is not another continuity family, not another continuation family, not another
governed communication family, not the remote-operator family, and not yet the
delegated worker export family.

It is the family that owns:

- one bounded writable-surface descriptor over a selected subtree or file-set;
- one bounded repo write lease over that selected writable surface;
- one explicit repo-surface admission posture over that same selected surface; and
- later restoration and hardening over that same selected writable-surface lineage.

The family therefore sits downstream of `V59` / `V60` / `V61`, sibling to closed
`V63`, and upstream of `V65`.

- `V59` owns continuity-region identity and persistence posture.
- `V60` owns continuation / residual-intent identity.
- `V61` owns governed communication ingress / egress and communication posture.
- `V63` owns remote operator session / view / response / control posture.
- `V64` owns bounded repo-bound writable-surface authority over the already shipped
  core.
- `V65` should later own delegated export / worker reconciliation under `V48`.

`V62` and `V63` remain sibling law here:

- connector posture remains separate sibling trust law
- remote operator posture remains separate sibling transport/control law
- neither becomes writable entitlement basis by default in `V64-A`

## 2. Writable-Surface Boundary Thesis

The central law is:

- writable surface is entitlement law;
- continuity region is persistence/context law;
- communication basis may contextualize or justify write posture, but is not itself
  the writable lease;
- remote control posture is not repo-surface authority;
- and writable-surface widening must not widen mutation semantics at the same time.

So:

- one continuity region is not one writable-surface lease;
- one prior successful write is not standing repo authority;
- one message or approval lineage is not repo-surface authority by itself;
- one remote operator response is not repo authority;
- one writable-surface lease is not shell authority;
- one writable-surface lease is not execute authority;
- one writable-surface lease is not dispatch authority;
- one writable-surface lease is not delegated worker authority; and
- all-repo writable authority is not selected in the starter slice.

## 3. Starter Artifact Ladder

The family ladder should add these new shapes over the shipped `V59` / `V60` / `V61`
surfaces:

1. `agentic_de_repo_writable_surface_descriptor@1`
   - typed writable-surface descriptor
   - one selected subtree or file-set only
   - explicit inclusion / exclusion basis
   - canonical normalized path-membership basis
   - explicit surface-scope summary
2. `agentic_de_repo_write_lease_record@1`
   - typed bounded repo write lease
   - explicit writable-surface descriptor ref
   - explicit continuity / continuation / communication consumption summary
   - explicit lease verdict family
   - explicit frozen-policy anchor
3. `agentic_de_repo_write_surface_admission_record@1`
   - typed repo-surface admission posture
   - explicit writable-surface descriptor ref
   - explicit write-lease ref
   - explicit per-target occupancy / admissibility basis
   - explicit preserved narrow write semantics summary
4. later restoration surface
   - restoration / reintegration over the same writable-surface lineage
5. later hardening surface
   - advisory hardening over the same writable-surface lineage

Those shapes should consume, not reopen, the shipped:

- `agentic_de_workspace_continuity_region_declaration@1`
- `agentic_de_workspace_continuity_admission_record@1`
- `agentic_de_workspace_occupancy_report@1`
- `agentic_de_workspace_continuity_reintegration_report@1`
- `agentic_de_loop_state_ledger@1`
- `agentic_de_continuation_refresh_decision_record@1`
- `agentic_de_communication_ingress_packet@1`
- `agentic_de_surface_authority_descriptor@1`
- `agentic_de_ingress_interpretation_record@1`
- `agentic_de_communication_egress_packet@1`

The starter slice is instantiated here only because broader writable semantics and
worker export are later.

That should be read as:

- instantiated_here:
  - one writable-surface descriptor only
  - one write lease only
  - one repo-surface admission posture only
  - one selected subtree or file-set only
  - the currently selected narrow local-write semantics only
- deferred_to_later_lane:
  - writable-surface restoration
  - writable-surface hardening
  - broader write-mutation semantics
- deferred_to_later_family:
  - delegated worker export / reconciliation
  - shell / execute authority
  - dispatch authority

## 4. Starter Runtime Boundary Model

The starter writable-surface path should be:

```text
shipped V59 continuity region + shipped V60 continuation posture + shipped V61 communication posture
  -> repo writable-surface descriptor
  -> repo write lease record
  -> later if one in-surface target is selected:
       consume explicit per-target occupancy / admissibility basis
       -> repo write surface admission record
  -> later restoration over that same writable surface
  -> later advisory writable-surface hardening
```

What changes here is not continuation law, not communication law, not remote control
law, and not worker export law.

What changes is that one repo work surface stops being an implicit consequence of one
exact exemplar path and becomes one explicit bounded writable-surface authority
object.

## 5. Starter Freeze

`V64-A` should stay frozen to one exact starter path:

- one selected writable-surface descriptor only
- one bounded repo write lease only
- one typed repo-surface admission record only
- one selected writable surface only:
  - one declared subtree or file-set
  - no all-repo authority
- one continuity-root consumption mode only:
  - shipped `V59` continuity-region identity and occupancy posture as basis only
- one continuation consumption mode only:
  - shipped `V60` continuation refresh posture as basis only
- one communication consumption mode only:
  - shipped `V61` lineage where write posture or approval lineage is relevant
- one preserved mutation-semantics posture only:
  - `local_write`
  - `create_new`
- no append / overwrite / rename / delete widening
- no shell / terminal control
- no execute widening
- no dispatch widening
- no delegated worker export or reconciliation
- no remote-operator law mutation

The family should not expand to broad repo-admin law, free-form shell authority, or
delegated worker authority in the first slice.

## 6. Surface And Lease Law

Writable-surface posture should remain explicit and replayable.

At minimum the starter family should preserve these laws:

- writable-surface descriptor derives from:
  - explicit selected subtree or file-set basis
  - canonical normalized path-membership basis
  - explicit inclusion / exclusion basis
  - frozen policy basis
- write lease derives from:
  - writable-surface descriptor
  - shipped continuity basis
  - shipped continuation basis
  - shipped communication basis where relevant
  - frozen policy basis
- continuity region is not writable-surface entitlement by itself;
- communication lineage is not writable-surface entitlement by itself;
- write lease is replayable:
  - same selected surface basis
  - same consumed shipped basis
  - same frozen policy
  - same lease verdict

Only the writable-surface descriptor and write lease may say that a repo work surface
is selected here.

## 7. Admission And Membership Law

Repo-surface admission should remain explicit and bounded.

At minimum the starter family should preserve these laws:

- repo-surface admission derives from:
  - writable-surface descriptor
  - write lease
  - explicit per-target occupancy / admissibility basis
  - preserved narrow write-semantics basis
  - frozen policy basis
- the lease alone is not sufficient entitlement for every path inside the surface;
- path membership must be canonical and fail closed on aliasing surprises;
- symlink, alias, and indirection surprises fail closed;
- repo-surface admission is not shell authority;
- repo-surface admission is not execute authority;
- repo-surface admission is not dispatch authority;
- repo-surface admission is not delegated worker authority;
- repo-surface admission is replayable:
  - same descriptor
  - same lease
  - same per-target basis
  - same frozen policy
  - same admission record

Only the typed admission record may define starter-slice in-surface repo write
posture.

The starter slice does not yet solve:

- broader mutation semantics over the selected surface
- restoration over the wider surface
- advisory hardening over the wider surface
- delegated worker export / reconciliation

Those belong to later `V64` or `V65` work.
