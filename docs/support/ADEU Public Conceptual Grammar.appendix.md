## A better translation protocol, now attachment-aware

The translation pipeline should now be:

1. **Source anchoring**
   Bind source spans in the long note.

2. **Form-role typing**
   Classify each clause into concept, contract, law, claim, projection, mapping, or residual.

3. **Role-specialized extraction**
   Run different extraction rules per form role.

4. **Closure-core assembly**
   Assemble concepts, contracts, laws, claims, tensions, and entitlements into one closure core.

5. **Projection generation**
   Generate narrative, inferential, and implementation projections under explicit profiles.

6. **Witness emission**
   Emit the translation/projection witness.

This is better than “summarize prose into structure,” because it preserves the fact that different clauses do different inferential jobs.
