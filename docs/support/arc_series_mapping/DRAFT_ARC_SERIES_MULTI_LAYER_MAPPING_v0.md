# Draft Arc Series Multi-Layer Mapping v0

Status: support-layer synthesis draft over the existing arc selector series.

Authority layer: support.

This document does not supersede any `docs/DRAFT_NEXT_ARC_OPTIONS_v*.md` selector,
lock, architecture, implementation mapping, closeout artifact, or shipped schema. It
records one coherent map of the arc series as of the closed `V67` family, then names
the minimal missing family seams needed before the repo can honestly treat recursive
self-improvement as a robust governed capability rather than a philosophical target.

Interpretive doctrine:

- horizon-sensitive terms such as `bounded`, `complete`, `closed`, `deferred`, and
  `forbidden` are read using `docs/DRAFT_INTENT_HORIZON_GLOSSARY_v0.md`;
- planning-boundary language here is a support-layer scope guard, not lock-equivalent
  permanent prohibition;
- planning-vs-lock authority transfer is governed by
  `docs/DRAFT_INTENT_AUTHORITY_LAYERING_NOTE_v0.md`;
- future seam promotion should follow `docs/DRAFT_FUTURE_SEAM_PROMOTION_RULES_v0.md`.

## Scope And Corpus

Primary corpus:

- `docs/DRAFT_NEXT_ARC_OPTIONS_v2.md` through
  `docs/DRAFT_NEXT_ARC_OPTIONS_v52.md`;
- family architecture and implementation mapping docs referenced by those selectors;
- committed evidence inputs under `artifacts/agent_harness/` through the `V67-C`
  evidence lineage;
- `ADEU Recursive Self-Improvement Policy.md`;
- the practical flow, contribution alignment, roadmap, recursive-coordinate, ANM, UX,
  resident-agent, and support docs that shape the arc-family boundaries.

Naming layers used below:

- selector version: `DRAFT_NEXT_ARC_OPTIONS_vN`;
- family arc: `V32` through `V67`;
- implementation/evidence arc: `vNext+N` and committed evidence inputs;
- slice: `Vnn-A`, `Vnn-B`, and so on.

The selector series is not a linear feature backlog. It is a branching dependency map
whose dominant movement is:

1. make authority, artifacts, and validation explicit;
2. make reasoning and workflow behavior observable;
3. make domain, UX, benchmark, and resident-agent surfaces typed;
4. widen execution only after the semantic and deontic boundaries exist;
5. return to documentation, ergonomics, and recursive self-improvement only when the
   substrate can carry those claims without laundering authority out of prose.

## Robustness Target

For this synthesis, a robust ADEU arc structure is one in which every future
self-improvement candidate can pass through these cuts without ambiguity:

1. source and series position are known;
2. the candidate is described in O/E/D/U terms;
3. integration class and evidence burden are explicit;
4. human-internal ratification authority is explicit;
5. experiment, integration, rollback, and containment are typed;
6. outcome review can promote, demote, or quarantine the change;
7. execution widening cannot bypass the above cuts;
8. operator-facing projections make the decision legible without minting authority.

The current repo covers many parts of that chain, but not the chain as one closed
recursive self-improvement loop.

## Cross-Layer Dependency Map

### Authority And Source Law

Primary families:

- `V32`, `V34`, `V37`, `V47`, `V48`, `V55`, `V66`.

Current strength:

- strong artifact, source, policy, and documentation posture;
- clear fail-closed discipline;
- explicit separation of support, planning, architecture, lock, and runtime authority.

Remaining weakness:

- no single self-improvement source graph binds selector docs, family arcs, evidence
  artifacts, policy candidates, and post-use outcomes into one canonical recursive
  improvement register.

### Self-Knowledge And Semantic Identity

Primary families:

- `V40`, `V45`, `V49`, `V50`, `V53`, `V54`.

Current strength:

- architecture IR, repo self-description, semantic forms, symbol audit, edge ledger,
  and history semantics all exist as explicit substrates.

Remaining weakness:

- they are parallel substrates. The repo can inspect itself in many ways, but does not
  yet have a single self-improvement cartography layer that says which view controls a
  specific integration decision.

### Assessment And Evidence

Primary families:

- `V39`, `V41`, `V42`, `V44`, `V46`, `V51`, `V52`, `V55`.

Current strength:

- the repo has structural reasoning assessment, applied benchmark projection, simulation,
  paper-domain interpretation, and constitutional coherence checks.

Remaining weakness:

- evaluation of external advancements against the recursive self-improvement policy is
  not yet a first-class artifact family.

### Runtime, Continuation, Communication, And Delegation

Primary families:

- `V33`, `V35`, `V48`, `V56` through `V65`.

Current strength:

- one governed resident loop, communication membrane, connector branch, remote operator
  branch, repo write authority branch, and delegated worker reconciliation branch are
  all closed.

Remaining weakness:

- broader dispatch, execute, multi-worker orchestration, and recurring integration
  execution are still deferred. The repo can govern narrow action; it cannot yet safely
  widen self-improvement execution.

### UX, Operator Cognition, And Human-Internalism

Primary families:

- `V36`, `V63`, `V66`, `V67`.

Current strength:

- surfaces are treated as governed projections with ergonomic adjudication rather than
  ad hoc UI.

Remaining weakness:

- there is not yet one self-improvement operator surface that lets the human inspect,
  ratify, defer, reject, or revise candidate integrations while preserving the human as
  an internal component of the ADEU loop.

### Deferred Existing Branches

These are real but not part of the minimal recursive self-improvement seam set:

- `V43` governed contest participation: `deferred_to_later_family`;
- broader ARC-AGI tournament or competition execution beyond closed `V42`:
  `deferred_to_later_family`;
- repo-wide ANM migration or markdown supersession beyond closed `V66`:
  `deferred_to_later_family`;
- broader UX governance reopening beyond closed `V36` and `V67`:
  `not_selected_yet`;
- direct shell, terminal, unrestricted execute, hidden cognition, or autonomous
  constitutional amendment: `forbidden_by_current_planning_scope` until separately
  selected by later lock-level authority.

### Not Separate Families In This Minimal Set

Several tempting seams are intentionally treated as slices or consumers of the eight
families above rather than as ninth or tenth missing families:

- formal proof mirrors belong under `V70` / `V71` unless a later lock promotes them
  from sidecar evidence to constitutional authority;
- external paper, repo, and tool assimilation belongs under `V69` intake plus `V72`
  contained integration, with `V52` and external contribution alignment as consumed
  shaping inputs;
- governed memory/history widening belongs under `V73` outcome review unless a later
  family selects new memory authority beyond closed `V54`, `V59`, and `V60`;
- broader product rollout belongs under `V74` projection only when it consumes typed
  decision artifacts and does not mint authority by interface placement;
- direct autonomous amendment is not a missing family here because the current policy
  keeps constitutional integration under explicit human ratification.

## Recommended Order

Default order:

1. `V68`: make the map canonical enough to support later recursive decisions.
2. `V69`: admit candidate advancements as typed O/E/D/U records.
3. `V70`: classify and evaluate candidates against explicit thresholds.
4. `V71`: make ratification and amendment authority explicit.
5. `V72`: make integration, containment, and rollback operational.
6. `V73`: make post-use review and promotion/demotion explicit.
7. `V74`: project the loop to a human-internal operator surface.
8. `V75`: only then widen dispatch/execute/multi-worker capability.

Do not select `V75` before `V69` through `V73`. Otherwise execution capacity would
arrive before self-improvement governance.

## Proof Sketch For Minimality

Claim: the eight missing families above are the minimal robust seam set at family
granularity.

Coverage:

- `V68` covers the source/series-position cut. Without it, the system has many local
  selector records but no typed answer to "where does this candidate sit in the arc
  lattice?"
- `V69` covers the describe/map cut from the recursive policy. Without it, external
  advancements enter through prose, PRs, papers, or support notes without a native
  O/E/D/U candidate artifact.
- `V70` covers the classify/evaluate cut. Without it, observation-only, experiment,
  operational, structural, and constitutional integrations are rhetorically distinct
  but not machine-visible as burdens.
- `V71` covers the decide/ratify cut and the human-internalism invariant. Without it,
  the human remains present as conversation or review, but not as a typed authority
  locus inside recursive amendment.
- `V72` covers the integration/rollback cut. Without it, the rollback invariant in the
  policy remains doctrine rather than a released execution surface.
- `V73` covers the review cut. Without it, recursive improvement can integrate but
  cannot later demote, quarantine, or promote based on real use.
- `V74` covers the operator-legibility cut. Without it, the repo may have typed
  artifacts but no humane governed projection for ratification and exception
  adjudication; that pressures the human toward symbolic approval.
- `V75` covers the execution-widening cut. Without it, the loop remains able to
  reason about self-improvement but cannot safely scale implementation beyond the
  already closed single-path write/export/delegation substrates.

Independence:

- each cut has a different authority owner and failure mode;
- merging any adjacent pair would collapse a boundary the repo has repeatedly kept
  separate: description vs evaluation, evaluation vs ratification, ratification vs
  execution, execution vs review, review vs operator projection, projection vs authority;
- splitting any proposed family further is possible at slice level, but not required
  at family level because each proposed family owns one coherent contract surface.

Optimality:

- the set is sufficient because every recursive self-improvement candidate can be
  represented as:
  `arc graph position -> candidate record -> class/evaluation -> ratification ->
  contained integration -> outcome review -> operator projection -> optional governed
  execution widening`;
- the set is necessary because omitting any family leaves one required policy step or
  invariant without a native artifact owner;
- the set is minimal at family granularity because no proposed family is only a
  convenience wrapper around another family. Each protects a distinct boundary that
  would otherwise be laundered through prose, UI, runtime capability, or after-the-fact
  closeout.
