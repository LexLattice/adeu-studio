## Gap

The repo no longer lacks:

- one lawful continuation / residual-intent kernel;
- one lawful governed communication membrane over the selected resident seam; or
- one advisory communication hardening surface over that same resident lineage.

The repo still lacks:

- one bounded connector-admission surface that says which external connector path is
  admitted into the governed communication membrane at all;
- one explicit law that connector transport is transport only, not authority;
- one explicit law that upstream external assistant output is typed bridge material,
  not native witness, not task-law compilation, and not execution authority;
- one explicit ingress / egress bridge over shipped `V61` communication packets for
  one selected external assistant path only;
- one explicit principal-typed connector posture that says which connector-carried
  principal is selected now and which connector-carried principals remain deferred;
- one bounded family posture that keeps connector-specific trust distinct from:
  - resident communication law already owned by `V61`
  - remote operator UX reserved for `V63`
  - repo-bound writable authority reserved for `V64`
  - delegated export / worker widening reserved for `V65`

So the repo still lacks the family that would let it say:

- this exact external connector path is admitted under this exact bounded connector
  posture;
- this exact connector-carried principal is selected here, and no broader
  connector-principal set is admitted by default;
- this exact external assistant traffic became this exact governed bridge packet;
- this exact bridge packet consumed shipped `V61` communication law rather than
  inventing its own communication semantics; and
- connector transport remained non-authorizing unless later families explicitly
  promoted it.

This new family would ask:

  communication membrane?
- how should external assistant traffic be bridged into and out of shipped governed
  communication packets without letting connector transport become a hidden office?

  explicit and replayable rather than ambient?

  traffic selected here does not silently become human-via-connector law?

So this family may constrain:
