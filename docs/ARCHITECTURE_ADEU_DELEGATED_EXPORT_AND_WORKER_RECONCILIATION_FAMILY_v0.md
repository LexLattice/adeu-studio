# Architecture: ADEU Delegated Export And Worker Reconciliation Family v0

Status: architecture / decomposition draft for `V65`.

Authority layer: architecture_decomposition.

This note does not authorize implementation by itself. It defines the bounded family
role that should widen from one shipped local writable-surface lineage and one
already released `V48` worker substrate into one explicit delegated export /
worker reconciliation family without letting local repo authority, worker carrier
identity, or prior delegated success become ambient export, shell, execute, or
all-repo authority.

## 1. Family Role

`V65` is not another continuity family, not another continuation family, not another
governed communication family, not another remote-operator family, and not another
local writable-surface family.

It is the family that owns:

- one bounded delegated export starter over one shipped local writable-surface
  lineage only;
- later one bounded worker reconciliation surface over that same exported lineage;
  and
- later one advisory delegation hardening surface over that same exported lineage.

The family therefore sits downstream of `V64`, consumes released `V48` worker law
as sibling closed substrate, and remains later than all transport/control families.

- `V48` owns delegated worker binding / compiled boundary / worker envelope /
  conformance / topology law.
- `V59` owns continuity-region identity and persistence posture.
- `V60` owns continuation / residual-intent identity.
- `V61` owns governed communication ingress / egress and communication posture.
- `V64` owns bounded local repo-bound writable-surface authority.
- `V65` owns only the bounded export / reconciliation bridge between shipped `V64`
  local writable authority and released `V48` worker law.

`V62` and `V63` remain sibling law here:

- connector posture remains separate sibling trust law
- remote operator posture remains separate sibling transport/control law
- neither becomes delegated export authority basis by default in `V65-A`

## 2. Export-Bridge Boundary Thesis

The central law is:

- local writable-surface admission is local entitlement law;
- released worker binding / compiled / envelope / topology surfaces are worker-law
  carrier identity;
- delegated export bridge law is neither of those by itself;
- communication basis may contextualize export posture, but is not itself export
  authority; and
- export widening must not widen dispatch, multi-worker orchestration, or shell
  semantics at the same time.

So:

- one local writable-surface lease is not one delegated export admission;
- one released compiled worker boundary is not one lawful export by itself;
- one prior successful delegated worker run is not standing export authority;
- one message or approval lineage is not delegated export authority by itself;
- one delegated export admission is not shell authority;
- one delegated export admission is not execute authority;
- one delegated export admission is not dispatch authority; and
- all-repo or all-worker delegated authority is not selected in the starter slice.

## 3. Starter Artifact Ladder

The family ladder should add these new shapes over the shipped `V64` and released
`V48` surfaces:

1. `agentic_de_delegated_worker_export_packet@1`
   - one typed delegated export starter only
   - one shipped local writable-surface lineage only
   - one released worker carrier lineage only
   - one selected worker topology lineage only
   - no implicit fan-out or multi-worker set expansion
   - explicit exported work summary
   - explicit exported-work membership basis
   - explicit local-scope carry-through summary
   - explicit worker-carrier basis ref or equivalent
   - explicit selected worker-topology basis ref or equivalent
   - explicit worker-carrier identity summary
   - explicit export verdict family
   - explicit frozen policy anchor
2. `agentic_de_delegated_worker_reconciliation_report@1`
   - one typed reconciliation or reintegration-over-worker-result posture only
   - explicit exported lineage ref
   - explicit worker result / conformance carry-through
3. `agentic_de_delegated_worker_hardening_register@1`
   - one advisory hardening register only
   - explicit evidence-vs-recommendation separation

Those shapes should consume, not reopen, the shipped:

- `agentic_de_repo_writable_surface_descriptor@1`
- `agentic_de_repo_write_lease_record@1`
- `agentic_de_repo_write_surface_admission_record@1`
- `agentic_de_repo_write_restoration_record@1` where relevant
- `agentic_de_repo_write_reintegration_report@1` where relevant
- `agentic_de_continuation_refresh_decision_record@1`
- `agentic_de_communication_ingress_packet@1`
- `agentic_de_surface_authority_descriptor@1`
- `agentic_de_ingress_interpretation_record@1`
- `agentic_de_communication_egress_packet@1`

And it should remain downstream of the already released `V48` worker substrate:

- released binding-profile carrier surfaces
- released compiled-boundary carrier surfaces
- released worker envelope surfaces
- released worker conformance surfaces
- released worker topology surfaces

The starter slice is instantiated here only because broader dispatch and multi-worker
orchestration are later.

That should be read as:

- instantiated_here:
  - one export starter only
  - one exported local writable-surface lineage only
  - one released worker carrier lineage only
  - one bounded exported work artifact only
- deferred_to_later_lane:
  - worker reconciliation
  - advisory delegation hardening
  - richer exported payload families
- deferred_to_later_family:
  - dispatch widening
  - multi-worker orchestration
  - shell / execute authority

## 4. Starter Runtime Boundary Model

The starter delegated export path should be:

```text
shipped V64 local writable-surface descriptor + shipped V64 local lease/admission
  + shipped V60 continuation posture + shipped V61 communication posture
  + released V48 worker carrier identity
    -> delegated export starter
    -> later worker reconciliation over the same exported lineage
    -> later advisory delegation hardening
```

What changes here is not local writable entitlement law, not worker-law ownership,
not continuation law, and not communication law.

What changes is that one already admitted local writable path stops being purely a
local authority object and becomes one explicit bounded candidate for export into one
already governed delegated worker substrate.

## 5. Starter Freeze

`V65-A` should stay frozen to one exact starter path:

- one typed delegated export starter only
- one shipped local writable-surface lineage only
- one released worker carrier lineage only
- one selected worker topology lineage only
- no implicit fan-out or multi-worker carrier set
- one exported work artifact or path summary only
- one exported-work membership basis only
- one local writable-surface consumption mode only:
  - shipped `V64` descriptor / lease / admission as basis only
- one worker-law consumption mode only:
  - released `V48` carrier identity as basis only
- one continuation consumption mode only:
  - shipped `V60` continuation posture as basis only
- one communication consumption mode only:
  - shipped `V61` lineage where export justification or answer routing is relevant
- no fresh local surface selection
- no fresh local lease issuance
- no fresh local target admission
- no fresh worker topology widening
- no broader dispatch semantics initiated here
- no worker execution semantics redefined here
- no dispatch widening
- no shell / terminal control
- no execute widening
- no multi-worker orchestration
- no connector / remote-operator law mutation

The family should not expand to broad repo-admin law, free-form shell authority,
dispatch law, or recursive worker orchestration in the first slice.

## 6. Export And Carrier Law

Delegated export posture should remain explicit and replayable.

At minimum the starter family should preserve these laws:

- delegated export derives from:
  - shipped local writable-surface descriptor / lease / admission basis
  - explicit exported work summary
  - explicit exported-work membership basis
  - released worker carrier identity basis
  - shipped continuation basis
  - shipped communication basis where relevant
  - frozen policy basis
- shipped `V64-C` advisory hardening evidence may shape or drift-guard export
  posture, but is not export entitlement by itself;
- local writable admission is not delegated export entitlement by itself;
- released worker carrier identity is not delegated export entitlement by itself;
- delegated export does not initiate broader dispatch semantics by itself;
- delegated export does not redefine worker execution by itself;
- delegated export is not reconciliation by itself;
- delegated export does not carry worker-result semantics yet;
- exported work must preserve the shipped narrow `V64` mutation subset;
- exported work may not launder extra local paths outside admitted local scope;
- delegated export is replayable:
  - same shipped local basis
  - same released worker basis
  - same exported work summary
  - same exported-work membership basis
  - same consumed continuation basis
  - same consumed communication basis where relevant
  - same frozen policy
  - same export verdict

Only the typed delegated export starter may say that one local writable lineage is
selected for exported delegated-worker posture here.

## 7. Reconciliation Law

Worker reconciliation should remain explicit and bounded when later selected.

At minimum the family should preserve these laws:

- worker reconciliation derives from:
  - delegated export starter
  - explicit worker result / conformance carry-through
  - explicit local lineage carry-through
  - frozen policy basis
- export alone is not reconciliation by itself;
- worker carrier identity alone is not reconciliation by itself;
- reconciliation is not fresh local writable entitlement;
- reconciliation is not fresh worker topology or dispatch law;
- reconciliation is replayable:
  - same export starter
  - same worker result basis
  - same local lineage basis
  - same frozen policy
  - same reconciliation output

Only the later typed reconciliation surface may define delegated worker result
carry-through for this family.

The starter slice does not yet solve:

- richer exported payload semantics
- worker result reconciliation
- advisory delegation hardening
- dispatch or multi-worker orchestration
- shell / execute authority

Those belong to later `V65` work.
