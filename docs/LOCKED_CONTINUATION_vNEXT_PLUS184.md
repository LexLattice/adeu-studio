## Purpose

Freeze the bounded advisory ANM adoption-hardening posture for `V66-C` so the
repo can assess the shipped exact `V66-A` / `V66-B` documentation-governance
lineage for compile / prose-boundary drift without silently widening governed
source discovery, document authority profile law, class-policy law,
migration-binding law, reader-projection authority, semantic-diff authority,
markdown supersession law, or `V47` ANM language / IR / checker ownership.

`vNext+184` authorizes docs plus the first `V66-C` implementation path over
existing repo-owned packages, but not fresh source-set widening, fresh
migration-binding authority, fresh reader-projection authority, fresh
semantic-diff baseline law, repo-wide markdown-to-ANM transition posture,
generated-reader promotion into source authority, semantic-diff promotion into
authority, prose-to-policy laundering, advisory-to-live promotion, or `V47`
language widening.

In `vNext+184`, the new `V66-C` outputs are advisory documentation-governance
surfaces only. They do not change current runtime behavior, current markdown
authority, shipped `V66-A` classification semantics, shipped `V66-B` migration /
projection / diff semantics, or generated-reader authority posture by default.

  - the same shipped exact ANM-native path only:
    - one shipped `V66-A` source-set manifest only
    - one shipped `V66-A` authority-profile set only
    - one shipped `V66-A` class-policy surface only
    - one shipped `V66-B` migration-binding profile only
    - one shipped `V66-B` reader-projection manifest only where selected
    - one shipped `V66-B` semantic-diff report only where selected
  - one new advisory surface pair only:
    - `anm_compile_report@1`
    - `anm_prose_boundary_notice_set@1`
  - one explicit non-equivalence law:
    - compile report output is not source authority by itself
    - compile report output is not migration authority by itself
    - compile report output is not reader-projection authority by itself
    - compile report output is not semantic-diff authority by itself
    - compile report output is not markdown supersession law by itself
    - prose-boundary notice output is not compiled policy by itself
    - prose-boundary notice output is not transition law by itself
  - one explicit ordering law:
    - `V66-C` is downstream of shipped `V66-A` and `V66-B`
    - it does not replace `V47` language / IR / checker / evaluator ownership
    - it does not replace current markdown authority
    - it does not replace shipped `V66-A` source-set / authority-profile /
      class-policy ownership
    - it does not replace shipped `V66-B` migration / projection / diff
      ownership
  - one explicit posture law:
    - compile report remains typed and replayable
    - prose-boundary notice set remains typed and replayable
    - same shipped `V66-A` basis plus same shipped `V66-B` basis plus same
      selected prose-boundary evidence plus same frozen policy yields the same
      advisory outcome
    - committed lane artifacts outrank narrative prose when the advisory outputs
      derive evidence basis
    - evidence basis remains distinct from advisory outcome
    - generated reader projection and semantic diff remain shaping inputs only
    - current markdown remains controlling unless explicit shipped transition law
      already says otherwise
    - advisory outcomes remain non-entitling and non-escalating by themselves
