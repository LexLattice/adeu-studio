Freeze the bounded ergonomic instantiation adjudication starter for `V67-A` so
the repo can add one explicit ergonomic rule-authority stack, one frozen
ergonomic component registry, one per-surface visibility contract, one finite
ergonomic candidate-projection profile table, one provenance-bearing ergonomic
case envelope, and one bounded ergonomic adjudication request/result pair over
the already released Morphic UX governance stack without reopening released UX
governance ownership, minting new regions or lanes, laundering platform presets
into hard law, inferring authority from screenshots, or pretending to solve
layout.

`vNext+185` authorizes docs plus the first `V67-A` implementation path over the
existing repo-owned `adeu_core_ir` package, but not bounded adjudication-engine
computation, runtime measurement harvesting, typed runtime bridge artifacts,
generic layout-solver authority, screenshot-led authority, scalar ergonomic
scores, runtime personalization, clinical user modeling, or mutation of the
released UX-governance diagnostics / conformance family.

In `vNext+185`, the new `V67-A` outputs are ergonomic artifact-language starter
surfaces only. They do not change current runtime behavior, released Morphic UX
semantics, released projection or interaction law, released diagnostics /
conformance law, or realized runtime surface behavior by default.
