The starting hypothesis is close, but not final.

Two primary recursive axes survives adversarial testing. Four levels per axis survives. Lifecycle/status as an overlay survives. ODEU as local per-node structure survives. The part that does **not** survive unchanged is the second axis as originally named.

The best ADEU base space is **not**:

`abstraction/specialization × governance/publicity/execution`

It is:

`binding_depth × institutional_force`

with **4 levels on each axis**, plus overlays for lifecycle, visibility/publicity, trust, and phase, and with **ODEU as mandatory local node structure** rather than a recursion axis.

The key falsification result is this: **publicity fails the axis test**. Governance and execution can be unified as a gradient of direct effect on action. Publicity cannot. Public/private exposure changes distribution posture, not recursive semantic kind.

## RECURSIVE_SCHEMA_SPACE_THEORY

The most suitable abstract space is a **constrained, fibered recursive lattice**, not a bare hierarchy and not a free Cartesian plane.

Formally:

`RecursiveSchemaSpace = ⟨BaseGrid, LocalFiber, Overlays, DerivationEdges⟩`

Where:

* `BaseGrid = BindingDepth × InstitutionalForce`
* `LocalFiber = node-local inferential structure, including ODEU`
* `Overlays = lifecycle, visibility, trust, phase, maturity`
* `DerivationEdges = witness-bearing lawful relations between nodes`

This matters because ADEU artifacts have two fundamentally different kinds of structure:

1. **What the artifact means internally**
2. **What kind of recursive object it is relative to other artifacts**

Those must not be collapsed.

A distinction deserves to become a **primary recursive axis** only if it passes all four tests:

1. It classifies a **stable semantic kind of node**, not just a momentary state.
2. It changes **lawful derivation or admissibility**.
3. It recurs broadly enough across the schema ecology to justify global coordinates.
4. It varies independently enough that collapsing it would cause category confusion.

Applying that test:

* **Binding depth** passes.
* **Institutional force** passes.
* **Lifecycle/status** fails: same node can move candidate → stabilized → deprecated without changing recursive kind.
* **Publicity/visibility** fails: same governing schema can be internal or public without changing what it is.
* **ODEU facet** fails: O/E/D/U are internal dimensions of every ADEU node, not alternative recursive locations.
* **Materialization** partially passes, but only near the instance edge; it is better handled as a constrained derivation pattern plus carrier kind than as a third base axis.

So the right picture is:

* a **2D recursive base grid**
* with **node-local semantic fibers**
* and **witness-bearing derivation edges**
* under **overlay state annotations**

That is the minimum space that prevents collapse without becoming overbuilt.

## SCHEMA_AS_INFERENTIAL_CONTAINER

A schema is not just a description. It is a **typed inferential container** that selects a scope and governs lawful transformations over that scope.

A minimal formal schema can be written as:

`S = ⟨K, Scope, Transforms, Invariants, Evidence, Deontics, Utility, Residuals, ODEU⟩`

Where:

* `K` is carrier kind
* `Scope` selects the in-scope objects or traces
* `Transforms` defines admissible transformations over in-scope items
* `Invariants` defines what must remain true
* `Evidence` defines what proof/provenance is required
* `Deontics` defines obligations, prohibitions, permissions, gates
* `Utility` ranks lawful options only
* `Residuals` captures deferred seams, ambiguities, external dependencies
* `ODEU` is the local semantic decomposition of the node

A transition is lawful only if:

* the object is in scope,
* the transform is admissible,
* invariants are preserved,
* evidence requirements are satisfied,
* deontic constraints are satisfied,
* residual seam rules are respected.

Utility never rescues an unlawful transition. It only ranks options **inside** the lawful subset.

### What changes for meta-schemas

When in-scope objects are themselves schemas, the same formal shape remains, but the domain lifts one level.

A meta-schema is still an inferential container, but its scope ranges over:

* schema nodes,
* schema families,
* schema derivation edges,
* schema promotion/amendment patterns,
* coordinate placement laws.

Its admissible transformations become operations such as:

* specialize
* bind
* project
* materialize
* observe
* amend
* ratify

So a meta-schema is not “different in kind” from a schema. It is the same inferential form, lifted over inferential containers.

## LOCAL_VS_RECURSIVE_STRUCTURE_DISTINCTION

This is the decisive separation.

### What belongs inside each schema node

Inside a node belongs everything that determines the node’s inferential content:

* carrier kind
* scope selector
* admissible transformations
* invariants
* evidence requirements
* deontic constraints
* utility ordering
* residual seams
* local ODEU decomposition

ODEU belongs here because every ADEU node is supposed to expose ontology, epistemics, deontics, and utility **simultaneously**.

If ODEU were a recursion axis, two bad outcomes follow:

* either each node must be forced into one dominant facet, which is false,
* or each node must be split into four pseudo-nodes, which destroys ADEU’s local semantic integrity.

So ODEU is not a coordinate. It is a **local fiber structure**.

### What belongs in the coordinate grid

Only distinctions that identify recursive kind across the schema universe belong in the base grid:

* binding depth
* institutional force

These answer two different questions:

* How bound is this inferential container?
* How much right does it have to constrain or participate in action?

### What belongs in overlays

Overlays are stateful or exposure-related annotations that can change without retyping the node:

* lifecycle/status
* visibility/publicity
* trust posture
* phase boundary
* maturity/posture tags

A node can remain at the same coordinates while moving from draft to stabilized, or from internal to public.

That is why lifecycle and visibility are overlays rather than axes.

Important nuance:

* **Visibility/publicity posture** is an overlay.
* **Access/confidentiality law** belongs in local deontics.

So “public/private” is not ignored. It is just not granted axis status.

### What belongs in derivation records

Derivation records are neither node-local semantics nor base coordinates. They are edge witnesses:

* parent/child relation
* derivation kind
* authority source
* conformance proof
* evidence/provenance
* amendment or ratification witness
* residual export or bridge witness

This distinction matters because not every derived artifact is a descendant in the same sense.

* specialization and binding produce descendants,
* projection can produce a faithful expression without new authority,
* observation can produce a record without descendance,
* ratification can increase force without increasing binding depth.

## CANDIDATE_COORDINATE_GRIDS

I tried to break the 2-axis proposal in six ways.

### 1. One-axis ladder

A plausible flattening is:

`meta -> family -> bounded -> instance -> live`

or a slightly richer 6-step chain.

This is tractable and easy to teach, but it fails the real ADEU test. It flattens two distinct questions into one order:

* how specialized is the schema?
* how much force does it have?

The moment you need to distinguish a descriptive family registry from a governing family contract, or an execution manifest from a live run, the one-axis chain either collapses categories or secretly smuggles in a second dimension.

Verdict: **under-dimensioned**.

### 2. Original 2-axis candidate

`abstraction/specialization × governance/publicity/execution`

This is close. It sees that two ladders are already present.

Its failure is not axis count but axis hygiene. The second axis mixes three different phenomena:

* authority
* exposure
* enactment

Authority and enactment can be unified. Exposure cannot.

Verdict: **nearly correct, but overloaded**.

### 3. Three-axis model

A serious alternative is:

`binding_depth × authority × materialization`

This is expressive. It can distinguish family-law, instance-manifest, and live runtime more finely.

The problem is practical optimality. The third axis matters mainly near the instance edge. Making it global inflates the whole space to solve a local problem. That produces sparse invalid cells, more classification burden, and weaker human usability.

Verdict: **expressive but over-dimensioned**.

### 4. Asymmetric 2-axis model

A tempting simplification is:

`binding_depth(4) × institutional_force(3)`

This is cleaner at first glance, but one of the force bands gets collapsed. The dangerous collapse is usually:

* observational + interpretive, or
* governing + operative

ADEU needs both distinctions.

Verdict: **too compressed on force**.

### 5. Lifecycle as a true axis

This fails the identity test.

A schema can move:

`candidate -> stabilized -> deprecated`

without changing what recursive kind it is. If lifecycle becomes an axis, ordinary maturation gets confused with coordinate relocation. That damages auditability and turns state change into pseudo-descendance.

Verdict: **category mistake**.

### 6. ODEU as a recursion axis

This fails the locality test.

ODEU is the internal semantic decomposition of every node. Turning it into a recursive axis would make “ontology-heavy” and “deontics-heavy” look like different recursive locations rather than different internal emphases within one ADEU node.

Verdict: **local structure mistaken for global coordinate**.

### Under-dimensioning vs over-dimensioning

Under-dimensioning creates:

* authority laundering
* manifest/live conflation
* descriptive/governing collapse
* weak audit trails

Over-dimensioning creates:

* sparse artificial cells
* false precision
* more opportunities for rhetorical misclassification
* higher human and machine burden

The winning grid is the **lowest-dimensional space that still blocks category collapse**.

## ADVERSARIAL_COMPARISON_OF_COORDINATE_SYSTEMS

The decisive test is whether real ADEU-style artifacts can be placed cleanly without conflation.

### Stress test 1: same binding depth, different force

At family depth, these are not the same kind of thing:

* `repo_schema_family_registry@1`
* `adeu_architecture_world_hypothesis@1`
* `adeu_architecture_semantic_ir@1`

They can all live at roughly the same binding depth, but they do not have the same force:

* registry is descriptive/observational
* world hypothesis is interpretive
* semantic IR is governing

A one-axis model cannot place this triad cleanly without hidden cross-labeling. A 4×3 force model tends to collapse observational and interpretive. The refined 4×4 force axis handles it exactly.

### Stress test 2: same boundedness, different force

These are both bounded, but not equally forceful:

* project-type grammar
* restricted operational schema

The first narrows interpretation or planning. The second constrains action. Same binding band, different force band.

This kills any attempt to collapse governance into specialization depth.

### Stress test 3: same instance depth, different force

These are both instance-level, but not the same recursive kind:

* execution manifest
* concrete implementation instance

The manifest binds one concrete context but remains pre-operative law.
The implementation instance is the live or completed operative realization.

That is why “instance” is not enough by itself, and why operative must remain distinct from governing.

### Stress test 4: same semantic node, different status

Take one governing task schema and change only:

* candidate
* accepted
* deprecated

The recursive type does not change. What changes is maturity overlay, sometimes with new witness records. That falsifies lifecycle as a primary recursion axis.

### Stress test 5: same semantic node, different publicity

Take one governing schema and vary:

* internal committed
* public exported

Its inferential content and recursive kind can remain the same. Publicity changes exposure and access posture, not recursive semantic genus. That falsifies publicity as a primary axis.

### Stress test 6: same node, O/E/D/U all present

Take a single ADEU semantic root. It contains ontology, epistemics, deontics, and utility in one node. There is no clean way to place it on an ODEU axis without either choosing a fake dominant facet or duplicating the node four times.

That falsifies ODEU as a recursion axis.

## OPTIMAL_RECURSIVE_COORDINATE_GRID

The optimal ADEU recursive base space is:

* **2 primary recursive axes**
* **4 levels per axis**
* **symmetric at the base**
* **constrained in lawful occupancy**
* **with lifecycle/visibility/trust/phase as overlays**
* **with ODEU as local node structure**

### Axis 1: Binding depth

This is the formalized version of “abstraction / specialization,” but sharper.

It asks:

**How much of the inferential container remains open, and how much is already bound?**

Levels:

0. `meta`
1. `family`
2. `bounded`
3. `instance`

Why four?

Because ADEU needs to distinguish:

* schema law over schema law
* reusable family grammar
* bounded project/task/operational class
* one concrete binding

Three levels collapses either `meta/family` or `bounded/instance`.
Five or more levels mostly re-encode carrier kinds or overlays.

### Axis 2: Institutional force

This is the corrected replacement for “governance / publicity / execution.”

It asks:

**How much right does this artifact have to shape, gate, or participate in action?**

Levels:

0. `observational`
1. `interpretive`
2. `governing`
3. `operative`

Why four?

Because ADEU needs to distinguish:

* descriptive record
* meaning-shaping/planning surface
* authoritative constraint surface
* live operative participation

Three levels collapses one of those distinctions.
Five or more usually starts leaking publicity, trust, or phase into the base grid.

### Why symmetry is still best

The base grid should stay symmetric at 4×4 because it is easier to reason over and easier to validate mechanically.

But lawful occupancy is **not** symmetric. For example:

* `operative` usually requires `instance`
* many `family × operative` cells are invalid or empty
* that is acceptable because this is a constrained lattice, not a promise that every cell must be populated

### Why lifecycle remains an overlay

Lifecycle changes should not force coordinate relocation.
Otherwise stabilization and deprecation become pseudo-descendants rather than state changes.

### Why ODEU remains local

ODEU is required everywhere, so it belongs inside every node.
It can be projected outward for analytics, querying, or visualization, but that does not make it a recursion axis.

## Final determination

The proposed **2-axis / 4-level** model is the best coordinate grid **only after one correction**.

Keep:

* 2 primary recursive axes
* 4 levels per axis
* lifecycle/status as overlay
* ODEU as internal per-node structure

Change:

* replace `governance/publicity/execution` with **institutional_force**
* move **publicity/visibility** out of the base grid and into overlays
* keep **access/confidentiality law** inside local deontics, not on the axis

So the final ADEU recursive base space is:

**`binding_depth × institutional_force`**

with:

* `binding_depth = {meta, family, bounded, instance}`
* `institutional_force = {observational, interpretive, governing, operative}`

This is the minimum space that is expressive enough to position ADEU schema artifacts, strict enough to constrain lawful derivation, simple enough for human and machine use, and disciplined enough to resist category collapse.
