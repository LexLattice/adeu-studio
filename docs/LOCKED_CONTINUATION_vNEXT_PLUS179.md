## Purpose

Freeze the bounded delegated export bridge posture for `V65-A` so the repo can
emit one typed delegated export packet over one shipped `V64` local
writable-surface lineage and one released `V48` worker carrier lineage without
letting local repo authority, worker carrier identity, communication reachability,
or prior delegated success become ambient export authority.

`vNext+179` authorizes docs plus the first `V65-A` implementation path over
existing repo-owned packages and thin backend seams, but not worker
reconciliation, advisory delegation hardening, broader dispatch semantics,
worker execution redefinition, multi-worker orchestration, shell or terminal
control, execute widening, all-repo delegation, advisory-to-live promotion, or
hidden-cognition governance.

In `vNext+179`, the new `V65-A` output is one delegated export bridge surface
only. It does not change local writable entitlement semantics, released `V48`
worker-law semantics, continuation semantics, governed communication law,
connector law, remote-operator law, or exit behavior by default.

    - `V65-A` is downstream of shipped `V64` and released `V48`
    - it does not replace local writable entitlement law
    - it does not replace released worker-law carrier identity

    - it does not replace shipped communication law
    - export bridge scope is widened before reconciliation is widened
    - worker reconciliation remains deferred to `V65-B`
    - advisory delegation hardening remains deferred to `V65-C`
  - one explicit posture law:
    - delegated export packet remains typed and replayable
    - same shipped `V64` basis plus same released `V48` basis plus same exported-work membership basis plus same consumed `V60` basis plus same consumed `V61` basis where relevant plus same frozen policy yields the same export output
    - exported-work membership is canonical, explicit, and fail-closed
    - exact local target / patch / artifact summary remains required
    - one released worker carrier lineage only is required
    - one selected worker topology lineage only is required
    - no result in this slice becomes dispatch authority, shell authority,
      execute authority, multi-worker authority, connector law, or remote law by default
    - no result in this slice widens mutation semantics beyond shipped
      `local_write/create_new`

  rather than reopen them:
  - local writable reuse is the default

  - `V62` connector posture remains sibling law, not delegated export basis
  - `V63` remote-operator posture remains sibling law, not delegated export basis
