## Purpose

Freeze the bounded `V67-B` adjudication slice so the repo can compute ergonomic
blocking posture, feasible-candidate posture, ordinal preference tiers, ambiguity
rows, and measurement-obligation rows over the shipped `V67-A` ergonomic artifact
language without reopening released UX governance ownership, minting new schema
families by default, introducing a generic layout solver, laundering utility into
constitutional law, inferring authority from screenshots, or promoting runtime
measurement evidence into sovereign adjudication.

`vNext+186` authorizes docs plus the first `V67-B` implementation path over the
existing repo-owned `adeu_core_ir` package, but not new ergonomic schema ids by
default, runtime measurement evidence artifacts, advisory runtime bridge reports,
runtime adaptive morph switching, screenshot-led authority, scalar ergonomic
scores, clinical user modeling, or mutation of the released UX diagnostics /
conformance family.

In `vNext+186`, `V67-B` computes only over the shipped `V67-A` surfaces and
emits bounded computed `ux_ergonomic_adjudication_result@1` outputs. It does not
change released Morphic UX semantics, released projection or interaction law, or
realized runtime surface behavior by default.

    - UX diagnostics / conformance mutation remains forbidden here
    - screenshot witness scoring remains forbidden here.

- deterministic adjudication helpers that:
  - validate request and consumed-basis lineage
  - apply hard-block reason families before advisory ranking
  - derive feasible candidates from the shipped finite candidate table only
  - emit blocked candidate rows, feasible candidate rows, ambiguity rows, and
    measurement-obligation rows over the shipped result schema
- no new ergonomic schema ids by default
- computed reference fixtures and reject fixtures for the bounded adjudication
  pass only
- tests that prove:
  - `report_status` remains artifact-validity-only
  - `overall_judgment` remains ergonomic-outcome-only
  - blocked candidates carry stable reason-code families
  - feasible candidates never bypass hard-law blocking
  - hard floors compose as max-plus-upward-user-preference only
  - inadmissible physical / visual chains block only dependent computations
  - preference tiers remain ordinal only
  - top-tier ties become explicit `needs_review` when deterministic ladder rules
    cannot distinguish them
  - source refs and hashes remain exact and replayable in computed outputs
- no runtime measurement evidence artifact lands in this slice
- no runtime bridge report lands in this slice
- no generic layout solver lands in this slice.
