## Decomposition Thesis

`V45-A` should not start with:

- whole-repo maximal cataloging;
- whole-repo symbol mining;
- whole-repo test-intent inference;
- optimization entitlement;
- recursive amendment binding.

The safest first move inside `V45-A` is:

- schema-family-registry first;
- schema-adjacent entity-catalog scaffolding second;
- bounded reconstruction appendix third;
- full repo widening later.

The current best branch-local hypothesis for that first move is:

```text
common core envelope
  + carrier overlays
  + lineage overlays
  + narrow named residuals
```

That hypothesis should remain:

- useful for the first slice;
- explicitly branch-local;
- not yet settled doctrine;
- not yet sufficient without reconstruction evidence.

In short:

```text
first schema registry kernel, then wider repo self-description
```

## Recommended First Concrete `V45-A` Slice

The first concrete slice inside `V45-A` should center on:

1. `repo_schema_family_registry@1` doctrine and row shape
2. minimal `repo_entity_catalog@1` scaffolding for schema-visible surfaces
3. one bounded representative reconstruction appendix over the schema corpus

That means the first slice should introduce at least:

- schema row identity and source path capture;
- family-cluster posture;
- primary carrier class;
- optional secondary role-form tags;
- lineage overlay;
- residual flags;
- classification posture and method;
- supporting evidence refs;
- minimal schema-entity entries inside the repo entity catalog;
- one bounded snapshot or fixture posture over the schema corpus.

The first slice should not try to finish:

- whole-repo entity coverage;
- symbol cataloging;
- open arc/slice dependency register release;
- test-intent mapping;
- optimization-register release;
- recursive-governance binding.

## Representative Reconstruction Appendix

The first slice should carry one bounded representative appendix over at least:

- `adeu_arc_task_packet`
- `adeu_architecture_analysis_request`
- `meta_loop_sequence_contract`
- `ux_interaction_contract`
- `adeu_core_ir`
- `policy_registry`
- `adeu.validator_result`

For each case, the appendix should show:

- extracted core envelope
- primary carrier class
- secondary role-form tags
- lineage overlay
- named residuals if present
- explicit reconstruction back to the concrete schema definition

This appendix is the main anti-laundering check on the current schema-core hypothesis.

## Family Scale Rules

To stay aligned with the `V45` branch, the first slice should obey these rules:

- treat the schema corpus as the first bounded subcorpus, not as the whole repo;
- keep entity-catalog widening subordinate to the schema-registry kernel in the first
  slice;
- classify with explicit precedence rather than naming-only heuristics;
- keep adjudication posture explicit;
- keep residual drift explicit rather than hiding it under generic escape hatches;
- emit descriptive findings only, not amendment entitlement;
- keep any Lean or formalization ambition downstream of the descriptive registry work.
