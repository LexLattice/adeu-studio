## Gap

The repo no longer lacks:

- one released semantic substrate family over `V49`;
- one released paper-domain consumer proof over `V52`;
- one normalized intake pack that tells the truth about the external
  history-semantics bundle and its blocker set.

The repo still lacks:

- one repo-owned history-semantics package that freezes the artifact-family ladder and
  schema/export posture under repo authority;
- one explicit source-authority decision for this family:
  - raw bytes authoritative; or
  - normalized text authoritative; or
  - dual raw-plus-normalized surfaces with one declared authority root;
- one bounded deterministic preclassification / ledger / slice / theme substrate whose
  heuristics are governed by regression coverage rather than ambient prototype taste;
- one explicit authority ladder that separates:
  - authority root artifacts
  - deterministic derived overlays
  - advisory reconstruction artifacts
  - advisory workspace synthesis;
- one clear rule for how O/E/D/U reconstruction and workspace snapshots remain
  advisory, never the source authority root.

The missing layer is therefore not merely another semantic substrate family, not a
paper-semantics widening, not a digest runtime bridge, and not yet a corpus-ingestion or
API/UI feature.

The missing layer is an ADEU history-semantics internalization / reconstitution family.

## Relationship To `V49`, `V52`, And The Imported Bundle

`V49`, `V52`, the external-prototype intake notes, and this new candidate family are
connected but non-identical.

`V49` asks:

- what is the released semantic identity and projection substrate that downstream
  families must reuse or narrow explicitly?

`V52` asks:

- how should ADEU own one bounded paper-domain semantic family over that substrate
  without silently minting a parallel semantic language?

The external-prototype intake notes ask:

- how should imported prototype bundles be preserved as evidence and normalized into
  repo-owned planning rather than treated as precedent-bearing overlays?

This new family asks:

- how should ADEU own one bounded history-semantics family instead of treating a
  heuristic prototype as ambient semantic authority?
- how should the family’s source artifact, ledger, slice, theme, O/E/D/U packet, and
  workspace layers be separated by authority level?
- how should source-authority semantics be decided explicitly before the first released
  contract?
- how should the current parser / grouping / lane-scoring heuristics be rebuilt under a
  regression corpus instead of imported ambiently?
- how should schema/export release posture be frozen so this becomes a repo-native
  typed artifact family rather than only a Python model cluster?

`V54-A` should therefore be read as:

- package-first;
- contract-first;
- explicit about source-authority semantics;
- explicit about schema/export release posture;
- explicit that the starter source domain is one normalized-text
  `conversation_history` class with full `User:` / `Assistant:` / `System:` headers;
- explicit that the only accepted timestamp prefix, when present, is one bracketed
  `YYYY-MM-DD HH:MM` prefix immediately before the full role header;
- explicit that `abstract_like` starter-domain widening remains `not_selected_yet`;
- explicit that the starter slice chooses the authority root and deterministic
  text-shape / preclassification overlay, not the whole advisory reconstruction stack
  at once.

`V54-B` should therefore be read as:

- later than `V54-A`;
- deterministic ledger / slice / theme substrate over the released source contract;
- explicit regression hardening for the parser, grouping, and preclassification layer;
- no API/UI/runtime widening.

`V54-C` should therefore be read as:

- later than released `V54-A` and `V54-B`;
- bounded advisory O/E/D/U packet release only;
- explicit that O/E/D/U packets are advisory reconstruction artifacts, not authority
  roots.

`V54-D` should therefore be read as:

- later than the source, ledger, and O/E/D/U core slices;
- one bounded workspace question / theme-frame / workspace-snapshot advisory seam only;
- explicit that any later Wave 0 or broader corpus lane remains deferred until the core
  history-semantics family is stable enough to consume such material lawfully.

So this family may constrain:

- one repo-owned adjacent package for history semantics;
- how released `V49` semantic identity and nearby semantic-family surfaces are consumed
  downstream;
- how source authority and advisory reconstruction are separated;
- how later workspace or corpus helpers are prevented from laundering soft heuristics
  into authority-root artifacts.

But it may not mint:

- ambient continuation inside `V49` or `V52`;
- silent promotion of the imported external package into precedent;
- a broad runtime, API, or UI feature in the first slice;
- authority-root claims from heuristic lane scoring alone; or
- implicit corpus-ingestion legitimacy from the current prototype tests alone.

Planning relationship:

- `V43` remains a valid connected candidate family from `v26`;
- `V49` remains a connected released family from `v32`;
- `V52` remains a connected released family from `v35`;
- `V53` remains a connected candidate family from `v36`;
- this draft introduces a twelfth connected candidate family rather than widening
  `V49` or `V52`.

## Why This Path

- It treats the imported bundle as serious shaping evidence without laundering it into a
  live continuation of nearby semantic families.
- It puts the source-authority decision before the wider advisory reconstruction stack.
- It separates artifact-family release from heuristic-layer adequacy.
- It makes schema/export posture a first-class blocker rather than a cleanup note.
- It leaves any future corpus, Wave 0, or runtime widening later.
