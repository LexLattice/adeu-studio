# Architecture: ADEU Resident-Agent Governed Communication Membrane Family v0

Status: architecture / decomposition draft for `V61`.

Authority layer: architecture_decomposition.

This note does not authorize implementation by itself. It defines the bounded family
role that should retrofit the repo’s already-real communication surfaces into a
governed communication membrane over the shipped `V60` continuation kernel, without
letting transcript, transport plumbing, or bridge posture become a hidden sovereign.

## 1. Family Role

`V61` is not another continuation family, not another effect family, not another live
harness family, not another continuity family, and not yet the connector or remote
operator family.

It is the family that owns:

- one bounded communication ingress packet over already-real repo surfaces;
- one bounded communication egress packet over those same surfaces;
- one explicit principal / speaker typing basis over ingress and egress surfaces;
- one explicit surface-authority classification surface;
- one explicit ingress-interpretation surface that converts raw message traffic into
  loop-relevant interpreted posture;
- one explicit resident bridge-office binding and rewitness / message-promotion gate
  in later slices;
- and one bounded communication posture law for when the loop owes status,
  clarification, authority request, escalation, or completion communication.

The family therefore sits downstream of `V60` and upstream of `V62` / `V63`.

- `V56` owns packet / proposal / checkpoint / ticket law for externally effective
  acts.
- `V58` owns live-turn origin discipline and reintegration over already shipped
  runtime paths.
- `V60` owns seed-intent compilation, residual-intent carriage, loop-state identity,
  and continuation posture.
- `V61` owns governed communication ingress / egress, interpretation, and later
  bridge-office posture over those already shipped surfaces.
- `V62` should later own connector-specific transport trust and admission.
- `V63` should later own remote operator UX and phone control-plane law.

## 2. Communication Boundary Thesis

The central law is:

- communication must be first-class and externally structured;
- it must not collapse into transcript memory, transport payloads, or UI send
  plumbing;
- it must not mint execution authority by default;
- and it must not replace `V60` continuation semantics.

In the current repo that means:

- raw message traffic must first become a typed communication ingress packet;
- communication interpretation must be explicit and replayable;
- outbound bridge communication must be typed rather than improvised directly from
  transcript state;
- message-to-witness promotion must stay explicit and bounded;
- and bridge-office posture must remain explicit rather than ambient.

So:

- raw transcript is not native witness;
- generic chat memory is not communication law;
- connector traffic is not automatically admitted communication law;
- UI send access is not bridge office;
- and hidden cognition is not the communication membrane.

## 3. Starter Artifact Ladder

The family ladder should add these new shapes over the shipped `V60` surfaces:

1. `agentic_de_communication_ingress_packet@1`
   - typed inbound communication artifact
   - bounded to already-real repo surfaces in the starter slice
   - explicit source principal class
   - explicit speaker class
   - explicit surface instance or session identity
2. `agentic_de_surface_authority_descriptor@1`
   - typed surface affordance / bounded authority posture over the active
     communication surface
   - explicit distinction among surface-carried affordances such as communication,
     observability, advisory, authority-request, execution, and witness-candidate
   - not message interpretation by itself
3. `agentic_de_ingress_interpretation_record@1`
   - typed and replayable interpretation over an ingress packet plus frozen policy
4. `agentic_de_communication_egress_packet@1`
   - typed outbound communication artifact over one bounded interpreted posture
5. later `agentic_de_bridge_office_binding_record@1`
   - explicit resident bridge-office binding for bridge-authorized outbound posture
6. later `agentic_de_message_rewitness_gate_record@1`
   - explicit message-promotion / rewitness gate when later slices select it

Those shapes should consume, not reopen, the shipped:

- `agentic_de_task_charter_packet@1`
- `agentic_de_task_residual_packet@1`
- `agentic_de_task_residual_refresh_packet@1`
- `agentic_de_loop_state_ledger@1`
- `agentic_de_continuation_decision_record@1`
- `agentic_de_continuation_refresh_decision_record@1`
- `agentic_de_live_turn_admission_record@1`
- `agentic_de_live_turn_reintegration_report@1`
- `agentic_de_workspace_continuity_reintegration_report@1`

The starter slice is instantiated here only because `V62` and `V63` are later.

That should be read as:

- instantiated_here:
  - one bounded governed communication starter over already-real repo surfaces
- deferred_to_later_family:
  - connector-specific transport trust law
  - remote operator UX law
- deferred_to_later_lane:
  - bridge-office binding
  - rewitness / message-promotion gate

## 4. Starter Runtime Boundary Model

The starter communication path should be:

```text
already-real repo message/send surface
  -> communication ingress packet
  -> surface-authority descriptor
  -> ingress interpretation record
  -> if the interpreted posture is loop-relevant:
       consume shipped V60 charter / residual / loop / continuation outputs
       -> derive one bounded communication egress posture
       -> emit one communication egress packet
  -> later V61 bridge-office and rewitness gates
  -> six-lane run summary
```

What changes here is not the inner act law, not the effect law, not the continuity
law, and not the continuation kernel.

What changes is that the loop stops treating communication surfaces as semantically
open pipes.

The communication membrane is therefore an explicit kernel output over already shipped
runtime/governance surfaces.

It is never a substitute for shipped `V60` continuation selection, and any outbound
communication posture must still be derived from the already ordered `V60` charter /
residual / loop / decision path rather than reordering it.

## 5. Starter Freeze

`V61-A` should stay frozen to one exact starter path:

- communication surfaces:
  - existing web copilot
  - existing GPT-5.4 desktop workbench
  - existing resident session send/message seams
- transport posture:
  - existing surfaces only
  - no connector-specific transport
  - no remote-operator UX
- starter shapes only:
  - one communication ingress packet
  - one surface-authority descriptor
  - one ingress interpretation record
  - one communication egress packet
- outbound posture only:
  - status update
  - clarification question
  - authority request
  - escalation notice
  - completion report
- authority posture:
  - communication only
  - not execution
  - not native witness by default
  - not repo-write authority

The family should not expand to connector transport, phone UX, bridge-office binding,
message rewitness, repo-bound writable authority, or execution widening in the first
slice.

## 6. Interpretation Law

Communication interpretation should remain explicit and replayable.

At minimum the starter family should preserve these laws:

- interpretation derives from:
  - typed ingress packet
  - surface-authority descriptor
  - frozen policy basis
  - shipped `V60` loop-state context where relevant
- interpretation is not hidden operator discretion;
- interpretation is not raw transcript echo;
- interpretation is replayable:
  - same selected ingress packet
  - same selected surface facts
  - same frozen policy
  - same interpretation posture

Only the interpretation record may decide whether inbound communication is treated as:

- clarification response
- authority response
- status request
- charter-amendment candidate
- advisory-only message

Ingress alone may not self-promote into task law or witness.

In the starter slice:

- `charter-amendment candidate` is communication posture only;
- it is not charter mutation;
- it is not residual mutation;
- it is not continuation mutation;
- and it does not reopen starter seed-ingress law.

## 7. Egress Law

Communication egress should remain explicit and bounded.

At minimum the starter family should preserve these laws:

- egress derives from:
  - interpreted communication posture
  - shipped `V60` continuation outputs
  - frozen policy basis
- egress is typed communication only;
- egress is not act authority;
- egress is not hidden bridge-office behavior;
- egress is replayable:
  - same selected interpretation
  - same selected continuation basis
  - same frozen policy
  - same egress posture

Only the communication egress packet may define the bounded outbound communication
artifact for the slice.

Transcript text alone may not self-promote into lawful egress.

## 8. Surface Authority Law

Surface classification should remain first-class.

At minimum the starter family should preserve these laws:

- a surface is classified explicitly;
- classification is replayable:
  - same selected surface facts
  - same frozen policy
  - same descriptor
- communication access is not permission;
- communication access is not office binding;
- transcript visibility is not native witness;
- execution or witness-candidate classification cannot be inferred implicitly from
  a send surface.

The descriptor therefore records surface affordance / bounded authority posture only.
It does not interpret individual messages by itself.

The starter slice may classify surfaces, but it may not yet promote messages into
native witness or office law.

## 9. Posture Law

The starter communication membrane should remain bounded to outcomes such as:

- `status_update`
- `clarification_question`
- `authority_request`
- `escalation_notice`
- `completion_report`
- `advisory_only_message`

Those outcomes should not be over-read.

In particular:

- they are communication postures only;
- they are not execution tickets;
- they are not charter mutation by default;
- they are not witness promotion by default;
- they do not replace `V60` continuation semantics.

The same selected communication evidence basis plus the same frozen policy must yield
the same communication posture.

## 10. Bridge Office Boundary

Resident Codex may remain both the operational actor and the communication bridge, but
that cannot remain ambient forever.

So the family should preserve this ladder:

- `V61-A`
  - bounded governed communication starter over already-real surfaces only
  - runtime may emit a governed egress packet over the resident surface
  - that posture is not yet explicit bridge-office behavior
  - no office binding yet
- `V61-B`
  - explicit resident bridge-office binding
  - explicit rewitness / message-promotion gate
  - explicit outbound bridge authority posture

That means:

- bridge posture may be instantiated later;
- it may not be assumed ambiently in the starter slice;
- and public access to a surface does not imply bridge office.

## 12. Deferred Seams

The starter slice should keep these seams explicit:

- deferred_to_later_lane:
  - bridge-office binding
  - rewitness / message-promotion gate
  - richer attachment or rich-media semantics
  - multi-party thread arbitration
- deferred_to_later_family:
  - connector-specific transport trust and external assistant bridge
  - remote operator UX and phone control-plane law
  - repo-bound writable authority
  - delegated export / worker reconciliation

So the family should be read as:

- instantiated_here:
  - governed communication packet starter over already-real repo surfaces
- not_selected_yet:
  - bridge-office binding and rewitness in `V61-B`
  - advisory communication hardening in `V61-C`
- deferred_to_later_family:
  - connectors
  - remote operator UX
  - repo widening
  - worker export
